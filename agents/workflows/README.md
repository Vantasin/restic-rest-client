# Agent Workflows Directory

This directory contains repeatable agent workflows for common tasks in the
repository.

## Files

- [`review.md`](./review.md): review index that routes to the appropriate review
  workflow
- [`agent-review.md`](./agent-review.md): review workflow for the agent layer
- [`repo-review.md`](./repo-review.md): review workflow for repository function
  and operator-facing behavior
- [`change-logging.md`](./change-logging.md): how to summarize code and
  operational changes consistently
- [`documentation.md`](./documentation.md): how to update human docs and agent
  docs without creating drift
- [`new-area.md`](./new-area.md): how to add a new repo area without leaving
  docs, verification, and routing behind

## Purpose

These workflows explain how to work in the repo. They do not replace the human
documentation in `Docs/`; they tell agents how to use and update it.
